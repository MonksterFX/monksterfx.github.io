project: working with socket.io

28 Sep 2020

CC or CoffeinChat#

In a project im working on I recently got asked how we could implement a chat by ourselfs. We work a lot with GCP in this Project an so the first tough was to implement it with the help of firebase.

But because of the ongoing debate about GDPR we skipped the easy way and tough about the critical stuff in a chat: The live reload aka pushing news to the client.

Instantly three technical solutions where noted on the whiteboard:

  • intervall polling
  • long polling
  • websockets

But before going deeper in the technologies, I remembered there was as great solution for pushing stuff to the client: socket.io

I never worked before with socket.io but there is a nice tutorial on their page: https://socket.io/get-started/chat/

making a usable solution#

Tutorials are nice but what can we really do with it to make a fun little project out of it. Making Instant Chatting as simple as possible.

  • Starting a chat by a shareable url. I really appreciate it the way jitsi does it.
  • Having Nicknames
  • Meeting random people
  • Destroying each chat after you leave the page/close the session (also means storing nothing in the backend)
